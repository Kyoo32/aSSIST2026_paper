

# 📄 CausalCompass 논문 분석 — 5회차

**Yi et al., arXiv 2602.07915, 2026년 2월 제출 (v2: 2026년 4월)**
*"CausalCompass: Evaluating the Robustness of Time-Series Causal Discovery in Misspecified Scenarios"*
저자: Huiyang Yi, Xiaojian Shen, Yonggang Wu, Duxin Chen, He Wang, Wenwu Yu
(Southeast University, Jilin University)

---

## 1. 기본 요약

### 논문이 풀려는 문제

시계열 인과 탐색(TSCD)은 머신러닝의 핵심 과제이지만, 검증 불가능한 인과 가정에 대한 의존과 기존 벤치마크의 강건성 평가 부재로 인해 광범위한 채택이 저해됩니다. 즉, 현존하는 인과 탐색 방법들은 데이터가 자신의 가정을 완벽히 충족한다는 전제 아래 만들어지는데, 실제 데이터는 항상 이 가정을 위반합니다. 그런데 "가정이 위반됐을 때 각 방법이 얼마나 망가지는가"를 체계적으로 평가한 벤치마크가 없었습니다.

### CausalCompass란?

모델링 가정 위반 하에서 시계열 인과 탐색 방법들의 강건성을 평가하기 위해 설계된 유연하고 확장 가능한 벤치마크 프레임워크입니다. 새로운 인과 탐색 방법을 제안하는 것이 아니라, 기존 방법들이 가정 위반 시나리오에서 어떻게 동작하는지를 체계적으로 측정하고 비교하는 평가 도구입니다.

---

## 2. 핵심 구조: 8가지 가정 위반 시나리오

CausalCompass는 VAR, LGC, VARLiNGAM, PCMCI, DYNOTEARS, NTS-NOTEARS, TSCI, cMLP, cLSTM, CUTS, CUTS+ 등 11개의 대표적인 TSCD 알고리즘을 평가합니다.

8가지 위반 시나리오는 다음과 같습니다:

| 시나리오 | 내용 |
|---|---|
| **비선형 메커니즘** | 인과 관계가 비선형 함수로 이루어진 경우 |
| **잠재 교란 변수** | 측정되지 않은 공통 원인이 존재하는 경우 |
| **측정 오류** | 관측값에 노이즈가 추가된 경우 |
| **비정상 효과** | 인과 효과가 시간에 따라 변하는 경우 |
| **결측 메커니즘** | 데이터 일부가 누락된 경우 |
| **혼합 데이터** | 연속/이산 변수가 혼합된 경우 |
| **Z-score 표준화** | 전처리로 표준화된 경우 |
| **추세와 계절성** | 비정상 시계열 패턴이 존재하는 경우 |

### 핵심 발견 4가지

**① 딥러닝 기반 방법이 전반적으로 가장 강건하다**

어떤 단일 방법도 모든 시나리오에서 최적 성능을 달성하지 못하지만, 다양한 시나리오에서 전반적으로 우수한 성능을 보이는 방법들은 거의 예외 없이 딥러닝 기반 접근법입니다.

**② NTS-NOTEARS는 전처리에 극도로 의존한다**

NTS-NOTEARS는 표준화 전처리에 극도로 의존하여, 기본 설정에서는 성능이 낮지만 표준화 후에는 강한 성능을 보입니다. 이는 당신의 논문과 직접적으로 연관되는 핵심 발견입니다 — 전처리가 NOTEARS 계열의 성능을 결정적으로 바꾼다는 것이 이미 실증됐습니다.

**③ 방법마다 취약한 시나리오가 다르다**

제약 기반 방법(PCMCI), 노이즈 기반 방법(VARLiNGAM), 점수 기반 방법(DYNOTEARS), 위상 기반 방법(TSCI), Granger 인과 기반 방법(LGC), 딥러닝 기반 방법(cMLP, cLSTM) 각각이 서로 다른 가정 위반 시나리오에서 서로 다른 강건성 프로파일을 보입니다.

**④ 하이퍼파라미터 민감도 분석도 포함**

기존 벤치마크에서 하지 않았던 하이퍼파라미터 민감도 분석을 포함하여, 방법의 실제 안정성까지 평가합니다.

## 3. 기존 연구와의 차이점

| 항목 | 기존 벤치마크 | CausalCompass |
|---|---|---|
| 대상 데이터 | i.i.d. 데이터 중심 | 시계열 특화 |
| 평가 시나리오 | 1~3개 가정 위반 | 8개 체계적 시나리오 |
| 메서드 범위 | 특정 계열만 | 6개 계열 11개 방법 |
| 전처리 효과 분석 | ❌ | ✅ (표준화, 정규화 포함) |
| 하이퍼파라미터 분석 | ❌ | ✅ |
| 확장성 | 고정 | Python 패키지로 공개 |

## 4. 한계

**① 차분(differencing) 전처리 미포함**
8가지 시나리오에 Z-score 표준화와 min-max 정규화는 포함되지만, 차분(differencing)은 포함되지 않습니다. 이것이 당신의 연구가 들어갈 수 있는 틈새입니다.

**② i.i.d. 전용 방법(NOTEARS)을 시계열에 적용하는 방법들이 포함**
NOTEARS는 원래 i.i.d. 데이터용인데 NTS-NOTEARS 형태로 시계열에 확장된 버전을 평가합니다. 이 과정에서 시계열 특성과 NOTEARS의 분산 민감성이 어떻게 상호작용하는지는 충분히 분리 분석되지 않습니다.

**③ 비선형 관측 변환(observation transformation)은 미포함**
8개 시나리오 모두 인과 메커니즘의 위반이지, 관측 과정의 위반이 아닙니다. "인과 구조는 정상이지만 관측이 비선형 변환을 거친" 케이스(=당신의 논문 설정)는 CausalCompass에 없습니다.
