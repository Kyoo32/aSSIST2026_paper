# 논문 출판 가이드
## Preprint부터 저널 게재까지

Version 1.0

---

# 목적

이 문서는 논문를 작성한 후 Preprint 공개부터 학술지 게재까지의 일반적인 절차를 정리한 가이드입니다.

특정 출판사나 저널에 한정되지 않으며,
Elsevier, Springer, IEEE, ACM, Nature 등 대부분의 학술지에 적용할 수 있습니다.

---

# 전체 흐름

```text
연구 아이디어
      ↓
실험 및 연구 수행
      ↓
논문 작성
      ↓
코드 및 데이터 정리
      ↓
GitHub 공개 (선택)
      ↓
Zenodo DOI 발급 (선택)
      ↓
Preprint 업로드
      ↓
학술지 제출
      ↓
동료심사(Peer Review)
      ↓
논문 수정(Revision)
      ↓
게재 승인(Accept)
      ↓
논문 출판
      ↓
Preprint 및 GitHub 업데이트
```

- [ ] Supplementary Materials

- [ ] Highlights

# 2. 연구 자료 정리

재현 가능한 연구(Reproducible Research)를 위해
연구 자료를 정리합니다.

예시

- 소스코드
- 데이터
- 설정파일
- 실행 예제
- 실험 스크립트
- README

권장 폴더 구조

```
project/
├── paper/
├── src/
├── data/
├── examples/
├── notebooks/
├── figures/
├── README.md
├── LICENSE
└── CITATION.cff
```

---

# 3. GitHub 공개 (선택)

소프트웨어나 분석 코드를 공개할 경우 권장됩니다.

체크리스트

- [ ] README 작성
- [ ] LICENSE 추가
- [ ] CITATION.cff 작성
- [ ] 설치 방법 작성
- [ ] 실행 예제 제공

완료 후 Release를 생성합니다.

예시

```
v1.0.0
```

---

# 4. Zenodo DOI 발급 (선택)

코드를 논문처럼 인용할 수 있도록 DOI를 발급받습니다.

절차

1. Zenodo 가입
2. GitHub 연동
3. Repository 활성화
4. GitHub Release 생성
5. DOI 자동 발급

장점

- 영구 보관
- DOI 제공
- 버전 관리
- 논문에서 인용 가능

---

# 5. Preprint 공개

## Preprint란?

동료심사(Peer Review)를 받기 전에 공개하는 논문입니다.

장점

- 연구 우선권 확보
- 빠른 공개
- 피드백 수집
- 조기 인용 가능

---

## 대표적인 Preprint 서버

| 분야 | 플랫폼 |
|------|---------|
| 컴퓨터공학, AI | arXiv |
| 환경·지구과학 | EarthArXiv |
| 생명과학 | bioRxiv |
| 의학 | medRxiv |
| 사회과학 | SSRN |

# 7. 학술지 제출

대부분의 저널은 온라인 투고 시스템을 사용합니다.

일반적으로 제출하는 파일

- 논문
- Cover Letter
- Figure
- Table
- Supplementary Materials
- Highlights (요구 시)
- Graphical Abstract (요구 시)

추가로 입력하는 정보

- 저자 정보
- 키워드
- 추천 리뷰어(선택)
- Funding
- Conflict of Interest
- Data Availability
- AI 사용 여부

---

# 8. 동료심사(Peer Review)

가능한 결과

- Accept
- Minor Revision
- Major Revision
- Reject

Revision 시

- Reviewer Comment 확인
- 논문 수정
- 변경 사항 표시
- Response Letter 작성

---

# 9. 게재 승인(Accept)

승인 후에는

- 최종 원고 확인
- Proof 교정
- 저작권 관련 문서 제출

확인 사항

- 저자 이름
- 소속
- 그림
- 참고문헌

---

# 10. 논문 출판

출판 후에는

- DOI 확인
- 온라인 공개 확인
- 인용 정보 확인

---

# 11. 출판 후 해야 할 일

추천 순서

- GitHub 업데이트
- Zenodo 업데이트
- Preprint 업데이트
- ORCID 업데이트
- Google Scholar 확인
- 개인 홈페이지 추가
- 연구실 홈페이지 추가

# Preprint 업데이트

논문가 출판되면

Preprint는 삭제하지 않습니다.

대신

다음 정보를 추가합니다.

```
Published in:

Journal Name

DOI:

https://doi.org/...
```

필요하다면 새로운 버전(v2, v3)을 업로드합니다.

# 참고

Preprint 정책, 저작권 정책, Accepted Manuscript 공유 정책은 출판사 및 저널마다 다를 수 있습니다.

반드시 투고하려는 저널의 Author Guidelines를 확인한 후 제출하는 것을 권장합니다.
